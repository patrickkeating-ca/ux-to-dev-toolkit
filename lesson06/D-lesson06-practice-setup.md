# Lesson 6 Project Setup Checklist

## Complete Setup Verification

Use this checklist to ensure your project repository is properly configured.

---

## Phase 1: Repository Foundation

- [ ] Created repository on GitHub
- [ ] Repository is private (if for work)
- [ ] Cloned/initialized repository locally
- [ ] Connected local to remote (`git remote -v` shows URL)
- [ ] Folder structure created
- [ ] Initial commit made
- [ ] Pushed to GitHub successfully

---

## Phase 2: Core Documentation

- [ ] README.md created with navigation
- [ ] PLATFORM.md created with tech details
- [ ] docs/PRD.md template created
- [ ] docs/decision-log.md created with first entry
- [ ] .gitignore created and committed
- [ ] All core docs display correctly on GitHub

---

## Phase 3: Collaboration Setup

- [ ] CONTRIBUTING.md created
- [ ] STRUCTURE.md created
- [ ] Pull request template documented
- [ ] Team members invited (if applicable)
- [ ] Collaboration guidelines established
- [ ] All team members can clone repository

---

## Phase 4: Workflow Verification

- [ ] Created at least one feature branch
- [ ] Made commits on feature branch
- [ ] Pushed feature branch to GitHub
- [ ] Created pull request
- [ ] Reviewed and merged PR
- [ ] Deleted branch after merge
- [ ] Completed full workflow successfully

---

## Phase 5: Content Addition

- [ ] Added sample content to docs/discovery/
- [ ] Created at least one persona or research doc
- [ ] Updated decision log with real entry
- [ ] Tested markdown preview
- [ ] All links work correctly
- [ ] Images/screenshots load properly (if added)

---

## Phase 6: Quality Checks

- [ ] README navigation links work
- [ ] No broken internal links
- [ ] No sensitive data committed
- [ ] .gitignore excludes correct files
- [ ] Folder structure matches documentation
- [ ] All markdown renders correctly

---

## Daily Workflow Test

Can you do these without looking at notes?

- [ ] Pull latest changes
- [ ] Create feature branch
- [ ] Make changes and commit
- [ ] Push branch
- [ ] Create PR
- [ ] Merge PR
- [ ] Clean up branch

---

## Team Collaboration Test (If Applicable)

- [ ] Teammate can clone repository
- [ ] Teammate can create branch
- [ ] Teammate can push their branch
- [ ] Teammate can create PR
- [ ] You can review their PR
- [ ] PR can be merged successfully
- [ ] No permission issues

---

## Success Criteria

Your repository is ready for real work when:

- [ ] Everything above is checked
- [ ] Repository structure makes sense
- [ ] Documentation is clear and helpful
- [ ] Workflow feels natural
- [ ] Team understands how to contribute
- [ ] You're confident using it for actual projects

4. Ask for help
5. Don't struggle alone
